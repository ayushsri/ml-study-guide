AI Architecture Guide

Tools vs
Agent Skills

Tools give agents hands. Skills give agents expertise. Here's how they differ — and how they work together.

🔧
Primitive

Tool

A single callable function that performs one discrete action and returns a result. Stateless and single-step.

  • One action, one result
  • Defined by its function signature
  • No reasoning or planning
  • Called explicitly by the agent
  • Stateless between calls
  • Examples: web_search, run_code, send_email
🧠
Knowledge Package

Agent Skill

A folder of instructions, scripts, and resources the agent loads on demand to gain specialized expertise.

  • Procedural knowledge + best practices
  • Defined by a SKILL.md file
  • Loaded when the task matches
  • Can bundle scripts, templates, references
  • Portable across agent products
  • Examples: pdf-processing, data-analysis, legal-review

Side-by-Side Comparison
Dimension 🔧 Tool 🧠 Agent Skill
WHAT IT IS A callable function / API A folder with SKILL.md + resources
PURPOSE Do one specific action Know how to approach a task domain
HOW INVOKED Agent calls it explicitly Agent loads it when description matches
CONTAINS Code logic, function signature Instructions, scripts, templates, references
STATEFUL? No — stateless per call Loaded into context, guides multiple steps
SCOPE Single operation Multi-step workflow expertise
PORTABILITY Tied to a specific platform/API Just files — works across agent products
AUTHORED BY Developers (code) Anyone — markdown instructions
EXAMPLE web_search(), send_email() pdf-processing, legal-review, data-analysis
ANALOGY A hammer A carpenter's expertise
How a Skill Gets Activated
1

Agent starts up THINK

Only loads the name and description of each available skill — nothing more. Context stays lean.

2

User sends a goal THINK

Agent reads the goal and matches it against available skill descriptions to decide what expertise it needs.

3

Skill is loaded LOAD SKILL

The full SKILL.md is read into context — giving the agent step-by-step procedural knowledge, best practices, and any relevant scripts or templates.

--- SKILL.md frontmatter ---
name: pdf-processing
description: Extract text, fill forms, merge PDFs.
--- Use when user needs to work with PDF files. ---
4

Agent uses tools as instructed ACT

Following the skill's instructions, the agent calls the right tools in the right order — e.g. read_pdf(), then summarize(), then send_email().

5

Goal complete ✓

The skill provided the expertise; the tools did the execution. Together they completed a multi-step task the agent couldn't do with either alone.

Based on the Agent Skills open standard — originally developed by Anthropic, now an open ecosystem.
